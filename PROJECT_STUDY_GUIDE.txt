==================================================
1. PROJECT TITLE
==================================================
Smart Learning Assistant using RAG and Generative AI

==================================================
2. PROJECT IDEA
==================================================
This project is an AI-powered academic platform designed for students.
It helps users:
- learn from uploaded PDF notes
- summarize content
- ask questions from documents using RAG
- generate quizzes
- review resumes
- prepare for interviews
- get career guidance
- debug code and receive optimized suggestions

The goal is to combine traditional web development with modern AI services
to build a practical educational product.

==================================================

Students often struggle with:
- large study materials

This project solves that by providing one platform where AI assists the student

9. AUTHENTICATION WORKFLOW
==================================================
The application now supports real authentication.

Register flow:
1. User enters name, email, and password
2. Backend checks whether email already exists
3. Password is hashed
4. User is stored in SQLite
5. Backend returns signed token + user details
6. Frontend stores session in localStorage

Login flow:
1. User enters email and password
2. Backend checks user record
3. Password is verified against stored hash
4. Signed token is returned
5. Frontend stores session

Protected APIs:
- upload
- files/recent
- dashboard/stats
- summarize
- ask
- quiz
- resume
- interview
- career
- code-debug

11. PDF LEARNING WORKFLOW
==================================================
When a user uploads a PDF:
1. Frontend uploads the file to backend
2. Backend saves the PDF in uploads/
3. pypdf extracts text from the PDF
4. Text is split into overlapping chunks
5. SentenceTransformers converts chunks into embeddings
6. FAISS creates a vector index
7. Index is saved in faiss_indexes/
8. File metadata is stored in SQLite

Benefits:
- user can reopen files later
- summaries can be regenerated
- questions can be asked again
- quiz can be generated again

==================================================
12. RAG WORKFLOW
==================================================
RAG means Retrieval-Augmented Generation.

In this project:
1. User asks a question about an uploaded PDF
2. Question is converted into an embedding
3. FAISS searches similar document chunks
4. Most relevant chunks are retrieved
5. Gemini receives:
   - user question
   - retrieved context
6. Gemini generates a grounded answer
7. Frontend displays the formatted result

Why RAG is important:
- reduces hallucination
- improves relevance
- makes answers document-based

24. SHORT VIVA ANSWERS
==================================================
Q. What is RAG?
A. RAG stands for Retrieval-Augmented Generation. It retrieves relevant document chunks first and then sends them to the language model to generate more accurate answers.

Q. Why use FAISS?
A. FAISS is efficient for similarity search over vector embeddings. It helps retrieve relevant text chunks quickly.

Q. Why use SentenceTransformers?
A. It converts text into embeddings that capture semantic meaning, which is required for vector search.

Q. Why use Gemini?
A. Gemini provides strong generative capabilities for summarization, question answering, and guidance generation.

Q. Why SQLite?
A. SQLite is lightweight, easy to set up, and suitable for small to medium academic projects.

Q. Why FastAPI?
A. FastAPI is fast, simple, modern, and ideal for building Python AI APIs.

Q. What makes this project different from a normal chatbot?
A. It uses uploaded documents, persistent storage, RAG retrieval, structured modules, and multiple academic support features.
